# 🏷️ The Architect v2.6 — Complete Tag Glossary

This glossary documents all tags used in The Architect system. Tags are categorized by function.

---

## 📋 CORE CONTENT TAGS

### Speculation & Flexibility
- `[SPECULATIVE]` — Content generated as possibility, not confirmed canon; user may override
- `[PLAYER-DEFINED]` — Content controlled by player choice, not GM fiat
- `[FLEXIBLE]` — Content that can change if narrative requires; not locked
- `[CONFIRMED CANON]` — Content locked as world truth; cannot be changed without flagging

### Genre & Tone
- `[INTENTIONAL TROPE]` — A genre trope used deliberately with awareness (e.g., chosen one structure, but subverted)
- `[TONAL OUTLIER]` — Content that deliberately breaks the established genre tone for contrast
- `[INTENTIONAL TONAL CONTRAST]` — Deliberate clash between element's tone and dominant world tone

## 🎲 PLAY HOOKS & ENGAGEMENT

### Hook Types
- `[PLAY HOOK]` — Content that creates immediate player engagement opportunity
- `[HOOK TYPE: Chase]` — Players must pursue or flee; time and space create pressure
- `[HOOK TYPE: Social]` — Players navigate charged relationship or public situation
- `[HOOK TYPE: Investigation]` — Players gather and interpret scattered information
- `[HOOK TYPE: Survival]` — Players manage limited resources under threat
- `[HOOK TYPE: Moral]` — Players choose between options that both cause harm
- `[HOOK TYPE: Heist]` — Players plan and execute covert operation
- `[HOOK TYPE: Negotiation]` — Players broker agreement between incompatible interests
- `[HOOK TYPE: Discovery]` — Players stumble into something not yet part of known tensions
- `[HOOK TYPE: Protection]` — Players shield someone/something from active threat
- `[HOOK TYPE: Escalation]` — Players decide whether to push situation further

### Hook Dependencies
- `[HOOK INDEPENDENT]` — Hook accessible without completing other hooks
- `[HOOK PREREQ: X — bypass: Y]` — Hook has prerequisite X but can be reached via bypass Y
- `[HOOK CHAIN DEPTH: X — bypass at step Y: Z]` — Chain depth documentation with bypass paths
- `[HOOK TYPE GAP: X]` — Hook type X has not appeared in recent batches; prioritize

### Session Hooks & Cliffhangers
- `[SESSION HOOK: type — thread: X]` — Session-ending hook (Revelation/Threat/Dilemma/Arrival/Discovery/Betrayal/Transformation)

## 📚 INFORMATION TIERS & ACCESS

### Information Classification
- `[PUBLIC]` — Anyone can know this; no barrier to access
- `[RUMORED]` — Circulates imprecisely; may be partially false
- `[RESTRICTED]` — Gatekept by faction, institution, or relationship
- `[HIDDEN]` — Actively suppressed or known only to specific individuals
- `[LOST]` — No living person can verify; requires archaeology/research

### Knowledge Distribution
- `[KNOWLEDGE PATH: Primary]` — Main access route to information
- `[KNOWLEDGE PATH: Secondary]` — Alternative access route
- `[CONTESTED KNOWLEDGE]` — Multiple sources give conflicting versions
- `[ROLEPLAY ACCESS: X — no dice required]` — Information accessible through in-character action X without skill check
- `[KNOWLEDGE BOTTLENECK: X]` — Single NPC monopoly on knowledge (violation — needs correction)

**Total Tags: 237+**

**Usage Principle**: Tags make world architecture visible to both the LLM and the GM. They are system metadata — rarely read aloud to players, but essential for maintaining world coherence across sessions. LLM Context Markers (`⚓`) serve a special role: they are memory anchors that prevent the LLM from losing track of critical facts during live sessions.
